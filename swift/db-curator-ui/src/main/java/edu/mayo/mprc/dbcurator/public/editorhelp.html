Curation Editor


Mayo Clinic Proteomics Resource Center (MPRC)


Purpose:  This application is used to create FASTA databases by performing a number of steps.  These steps will either add, remove, or modify sequences.  Steps will happen in order so that if you add a sequence in a previous step it may be removed or modified by a subsequent step.



Use:
  • There are a number of things that should be included in all curations.  These include:
    • ID:  this is a read only field that shows the id of the curaiton that is displayed.  A curation acquires and ID after it is saved.  This ID can later be used to retrieve the curation.
    • Unique Name:  A unique name that must not be longer than 21 characters and cannot contain spaces
    • Curation Title:  A fairly brief description of the curation
    • Owners Email:  the email of the person who should be notified with regards to this curation such as when it gets refreshed (see below)
    • Next automatic refresh date:  the date that the curation should next be automatically re-run (see below) (Optional).
    • Re-run expiration date:  when should the curation should not be re-run after re-run (Optional).
    • How often to re-run: the frequency that a re-run should be performed
    • Any notes to make:  A place to put more detailed information about the curation
  • Progress report:  this box is used to display any messages to the user as well as show the progress of an executing curaiton.
  • Load Curation: clicking this will prompt the user for different option to load a curation
    • enter an id of an existing curation to load
    • create a new curation with no set properties
    • create an editable copy of the curation that is currently open.  If this is done then a shortname then the unique name will not be set and needs to be re entered.
    • cancel - just leave the current curation open
  • Save: clicking this will save the curation so it can be accessed later.  The ID for the curation will then be displayed.  Note that a curation cannot be saved after it has been run
  • Test: (not yet implemented) performs a complete run of the curation but leaves no trace of the run meaning that no files are retained and the curation is not saved
  • Run:  performs the curation and generates the resulting .fasta file and then saves the curation when it is complete.  Not that editing of a curation after running is not permitted.
  • Choose a step to add:  this list gives the steps that can be added to the curation.  Steps will be added at the end (bottom) of the curation and then you can move them to where you would like them in the process.
  • Steps to run:  this area show the steps that are currently part of the curation.  You can move them up and down or remove them from the curation using the respective buttons.
  • Path to completed curation file:  this text box can be copied and after the curation is run the path to the file will be located
Step Types (more steps will be added upon user request):
  • New FASTA database from FTP 
    • specify a ftp address for accessing a FASTA database over anonymous ftp
    • you can alternatively select from a list of commonly requested databases (if you have one you use a lot let us know).
    • if you want to always use the most up-to-date version then check the "Always check for newest release" box otherwise any re-runs of the curation will just use the same fasta file if it can.
  • FASTA header filter - filter the database based on either simple text or regular expression criteria
    • This step allows you to enter some words to search for in the header of each FASTA sequence.  The "Simple Text" setting will just take a space delimitted set of words and to see if they exist in the sequence.  You must specify an option
      • "Any" - meaning that if the header contains any of the given words then you should include that sequence
      • "All" - meaning all of the words given must be in the header
      • "None" - meaning that none of the words given can appear in the header
    • This step also allows you to enter a regular expression that you want to match in the header.  The "All" and "Any" setting work identically in this case and if you want to specify multiple regular expressions then you should add another step to do this.  The "None" setting will invert the results meaning that you don't want to match the given regular expression.
  • Sequence randomization - take the sequences and either reverse or scramble them in order to perform some statistical tricks
    • This performs an operation on the sequences to provide a nonsense sequence.
      • Sequence Reversal - takes each sequence and reverses it.  Each header is appended with <REVERSED>.
      • Sequence Scamble - takes each monomer in the sequence and randomly moves it to a new location.  Each header is appended with <SCRAMBLED>
      • You can choose to retain the original sequences as well in which case both the original and manipulated sequences will be included.
  • Add a manual sequence - Enter a header and a sequence of your own to the final database.  Anything in the "Header" will be appended with a '>' character if one is not already there.  If you do not enter a header then the first line in the sequence must be the header (with '>') which will be automatically detected and moved to the header location.
